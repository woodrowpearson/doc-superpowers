## Plan Template

Audit always writes a report file. This is the structured handoff to `update`:

```markdown
# Documentation Update Plan — {date}

**Generated by:** /doc-superpowers audit
**Branch:** {current branch}

## Scope
{which scopes were audited}

## Findings Summary
| Priority | Count | Action |
|----------|-------|--------|
| P0 Critical | {N} | Must fix before merge |
| P1 Stale | {N} | Update recommended |
| P2 Incomplete | {N} | Add coverage |
| P3 Style | {N} | Low priority |

## Update Tasks

### P0 — Critical Updates
- [ ] {doc path}: {description of what needs changing}

### P1 — Stale Updates
- [ ] {doc path}: {description}

### P2 — Missing Coverage
- [ ] {doc path}: {description}

## Execution
Run `/doc-superpowers update` to auto-apply fixes from this plan.
```

Save to `docs/plans/{date}-audit-report.md`. The `update` action reads the most recent `*-audit-report.md` file as its input when no in-session audit is available.
